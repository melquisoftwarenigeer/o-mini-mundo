Sim, o PHPUnit é a escolha recomendada para testes no Laravel. O Laravel já vem com o PHPUnit configurado por padrão, e ele oferece uma integração muito boa com a estrutura do framework.

Aqui estão alguns motivos para usar o PHPUnit no Laravel:

  1. Integração com o Laravel: O Laravel oferece uma série de helpers e funcionalidades que tornam o uso do PHPUnit bem simples. Ele já vem com arquivos de configuração prontos, como o phpunit.xml, e você pode rodar os testes diretamente usando os comandos Artisan, como php artisan test.

  2. Testes Unitários e Funcionais: O PHPUnit no Laravel permite escrever tanto testes unitários quanto testes funcionais (de integração), com uma sintaxe limpa e fácil de entender. O framework oferece recursos como DatabaseMigrations e DatabaseTransactions para garantir que seu banco de dados esteja em um estado controlado durante os testes.

  3. Testes de API: Se você estiver desenvolvendo uma API, o Laravel oferece ferramentas como assertJson, assertStatus e assertJsonFragment, que se integram diretamente com o PHPUnit, facilitando a criação de testes para suas rotas e controladores.

  4. Cobertura de Testes: O PHPUnit permite gerar relatórios de cobertura de código, o que ajuda a garantir que seu código está bem testado e livre de bugs.

  5. Ferramentas adicionais: Além do PHPUnit, o Laravel também possui pacotes como o Laravel Dusk para testes de navegador e o Laravel Pest (uma alternativa mais fluida e concisa ao PHPUnit), mas, se você já está confortável com o PHPUnit, não há necessidade imediata de mudar.

Se você já está familiarizado com o PHPUnit, ele se adapta muito bem ao fluxo de trabalho do Laravel, então pode continuar utilizando-o sem problemas. No entanto, se quiser explorar alternativas, o Laravel também oferece boas opções como o Pest para uma sintaxe mais expressiva e concisa.

Passo 1: Criar um Projeto Laravel (se você ainda não tiver um)

Se você ainda não tem um projeto Laravel, o primeiro passo é criar um:

bash
composer create-project --prefer-dist laravel/laravel nome-do-projeto

Depois, acesse o diretório do projeto:

bash
cd nome-do-projeto

Passo 2: Configurar o PHPUnit (caso não esteja configurado)

O Laravel já vem com o PHPUnit configurado por padrão, mas caso você precise verificar ou modificar a configuração, o arquivo phpunit.xml estará na raiz do seu projeto.

Passo 3: Criar um Teste de Exemplo

Agora, vamos criar um teste básico para ver como funciona na prática. Vou te guiar em como fazer isso:

  1. Criar uma rota simples:

    Abra o arquivo routes/web.php e adicione uma nova rota:

    php
    Route::get('/saudacao', function () {
    return response()->json(['message' => 'Olá, Laravel!']);
});

  2. Criar um Teste:

    Execute o comando abaixo para criar um teste básico:

    bash
    php artisan make:test SaudacaoTest

    Isso vai criar um arquivo de teste em tests/Feature/SaudacaoTest.php.

  3. Escrever o Teste:

    Abra o arquivo tests/Feature/SaudacaoTest.php e modifique ele para verificar a resposta da nossa rota. O código ficaria assim:

    php
    <?php

namespace Tests\Feature;

use Tests\TestCase;

class SaudacaoTest extends TestCase
{
    /**
     * Testa a resposta da rota /saudacao.
     *
     * @return void
     */
    public function test_saudacao()
    {
        // Envia uma requisição GET para a rota /saudacao
        $response = $this->get('/saudacao');

        // Verifica se a resposta foi bem-sucedida e se a estrutura do JSON está correta
        $response->assertStatus(200)
                 ->assertJson([
                     'message' => 'Olá, Laravel!'
                 ]);
    }
}

    Aqui estamos usando o método get() para fazer uma requisição HTTP para a rota /saudacao, e depois fazemos as asserções de que o status da resposta é 200 e o conteúdo do JSON possui a chave message com o valor Olá, Laravel!.

Passo 4: Rodar os Testes

Com o teste pronto, basta rodá-los! Para isso, você pode usar o comando do PHPUnit ou o comando do Artisan, como mostrei antes:

bash
php artisan test

Ou, se preferir usar o PHPUnit diretamente:

bash
./vendor/bin/phpunit

Passo 5: Verificar o Resultado

Se o teste passar, você verá uma saída como esta:

bash
OK (1 test, 1 assertion)

Caso algo dê errado, o PHPUnit irá exibir um erro detalhado para você corrigir.

O que podemos fazer a seguir:

  • Criar mais testes para outras funcionalidades (como validações, controle de autenticação, etc.).

  • Adicionar testes de integração, verificando interações mais complexas com banco de dados e outros serviços.

  • Experimentar com testes de API usando o assertJson e outros helpers do Laravel.

Que ótimo que você quer avançar! Vamos construir uma suíte de testes para uma API protegida por JWT com um banco PostgreSQL. Esse é um cenário bem comum, e o Laravel tem várias ferramentas para facilitar o processo de teste em APIs com autenticação e banco de dados.

Vamos seguir um passo a passo bem estruturado. Aqui estão os pontos que vamos cobrir:

  1. Configurar a autenticação JWT (via tymon/jwt-auth).

  2. Configurar o banco de dados PostgreSQL para os testes.

  3. Criar a API protegida com JWT.

  4. Criar e rodar testes automatizados.

Passo 1: Instalar o tymon/jwt-auth para autenticação JWT

O pacote mais popular para autenticação JWT no Laravel é o tymon/jwt-auth. Vamos instalá-lo:

bash
composer require tymon/jwt-auth

Depois de instalar o pacote, publique a configuração:

bash
php artisan vendor:publish --provider="Tymon\JWTAuth\Providers\LaravelServiceProvider" --tag=config

Isso criará o arquivo de configuração config/jwt.php.

Agora, gere a chave secreta do JWT:

bash
php artisan jwt:secret

Essa chave será usada para assinar os tokens JWT gerados.

Passo 2: Configurar o Banco de Dados PostgreSQL para os Testes

No arquivo .env, configure o banco de dados PostgreSQL para os testes. Certifique-se de ter um banco PostgreSQL configurado. Exemplo de configuração:

env
DB_CONNECTION=pgsql
DB_HOST=127.0.0.1
DB_PORT=5432
DB_DATABASE=nome_do_banco
DB_USERNAME=usuario
DB_PASSWORD=senha

Além disso, no arquivo phpunit.xml, adicione o seguinte para garantir que o banco de dados seja restaurado para um estado limpo antes de cada execução de teste:

xml
<env name="DB_CONNECTION" value="pgsql"/>
<env name="DB_DATABASE" value="nome_do_banco_test"/>
<env name="DB_USERNAME" value="usuario"/>
<env name="DB_PASSWORD" value="senha"/>

No config/database.php, certifique-se de que a conexão pgsql está configurada corretamente para o ambiente de testes.

Passo 3: Criar a API Protegida por JWT

Agora, vamos criar um exemplo de API protegida que você precisará acessar com um token JWT.

  1. Criar a Rota e o Controlador:

    No arquivo routes/api.php, adicione uma rota protegida por JWT:

    php
    Route::middleware('auth:api')->get('/user', 'UserController@getUser');

  2. Criar o Controlador:

    Execute o comando para criar o controlador UserController:

    bash
    php artisan make:controller UserController

    No controlador, adicione o método getUser para retornar informações sobre o usuário autenticado:

    php
    <?php

namespace App\Http\Controllers;

use Illuminate\Http\Request;
use JWTAuth;

class UserController extends Controller
{
    public function getUser(Request $request)
    {
        $user = JWTAuth::parseToken()->authenticate();
        return response()->json(compact('user'));
    }
}

  3. Configurar o Guard JWT:

    No arquivo config/auth.php, defina o guard api como jwt:

    php
    'guards' => [
    'api' => [
        'driver' => 'jwt',
        'provider' => 'users',
    ],
],

  4. Criar o Middleware:

    Para garantir que a autenticação JWT seja aplicada corretamente, adicione o middleware auth:api nas rotas que deseja proteger. Já fizemos isso na nossa rota api.php.

Passo 4: Criar os Testes de API Protegida com JWT

Agora que nossa API está protegida com JWT, vamos criar testes para ela.

  1. Criar um Teste de Autenticação:

    Vamos criar um teste que faça login e obtenha o token JWT, e depois acesse a rota protegida com esse token.

    Execute o comando para criar um novo teste:

    bash
    php artisan make:test UserApiTest

    No arquivo tests/Feature/UserApiTest.php, adicione o seguinte código:

    php
    <?php

namespace Tests\Feature;

use Tests\TestCase;
use App\Models\User;
use Tymon\JWTAuth\Facades\JWTAuth;
use Illuminate\Support\Facades\Hash;
use Illuminate\Foundation\Testing\RefreshDatabase;

class UserApiTest extends TestCase
{
    use RefreshDatabase;

    /**
     * Testa a autenticação e a resposta da API protegida por JWT.
     *
     * @return void
     */
    public function test_user_can_access_protected_route_with_jwt()
    {
        // Cria um usuário para autenticar
        $user = User::factory()->create([
            'password' => Hash::make('senha123')
        ]);

        // Realiza o login e obtém o token JWT
        $token = JWTAuth::fromUser($user);

        // Faz uma requisição GET na rota protegida passando o token
        $response = $this->get('/api/user', [
            'Authorization' => 'Bearer ' . $token,
        ]);

        // Verifica se a resposta foi bem-sucedida e contém os dados do usuário
        $response->assertStatus(200)
                 ->assertJson([
                     'user' => [
                         'id' => $user->id,
                         'name' => $user->name,
                         'email' => $user->email,
                     ]
                 ]);
    }

    /**
     * Testa se a autenticação falha sem o token JWT.
     *
     * @return void
     */
    public function test_user_cannot_access_protected_route_without_token()
    {
        // Faz uma requisição GET na rota protegida sem o token
        $response = $this->get('/api/user');

        // Verifica se a resposta é uma falha de autenticação
        $response->assertStatus(401);
    }
}

  2. Rodar os Testes:

    Agora, basta rodar os testes:

    bash
    php artisan test
